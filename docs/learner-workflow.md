# Learner Workflow

### Purpose

This document defines how LinuxLabs should be run.

The learner should focus on solving the lab. Documentation, formatting, evidence organisation, and GitHub upload are handled after the lab is solved.

### Lab Duration Rule

Each lab must be designed to take **60–90 minutes maximum** for the active solving portion.

If a lab cannot realistically be completed within 90 minutes, it must be split into smaller labs rather than expanded into a large multi-hour exercise.

The timebox covers the learner's hands-on work: reading the scenario, planning, implementing, troubleshooting, verification, and capturing useful evidence. Final documentation and GitHub upload are handled separately after the learner provides evidence and reflection answers.

### Workflow

1. A lab is created in the correct topic folder.
2. The learner reads the scenario, requirements, constraints, and tasks.
3. The learner solves the lab on the target system within the 60–90 minute timebox.
4. The learner provides command output, observations, issues, and final verification evidence.
5. Seven reflection questions are asked.
6. The learner answers the seven reflection questions.
7. The final lab write-up is completed and uploaded to the correct folder.

### Learner Responsibilities

The learner is responsible for:

* understanding the scenario
* attempting the implementation
* running commands on the lab system
* capturing useful output
* identifying issues or blockers
* performing verification checks
* answering seven reflection questions

### Documentation Responsibilities

The documentation should capture:

* lab summary
* estimated duration and actual duration where known
* scenario
* requirements
* constraints
* assumptions
* implementation tasks
* key commands used
* files created or changed
* verification evidence
* RHEL vs Ubuntu comparison where relevant
* break/fix outcome
* post-reboot verification
* AI-assisted operations notes where relevant
* issues encountered
* decisions made
* security and production considerations
* final outcome
* reflection summary
* production improvements
* references used
* completion checklist

### Seven Reflection Questions

Ask exactly seven questions after the learner completes the lab:

1. What problem did this lab simulate?
2. What commands, files, or services mattered most in solving it?
3. What evidence proves your solution worked?
4. What broke, confused you, or required troubleshooting?
5. What would be risky about doing this in production?
6. Where did AI help, if at all, and what did you verify yourself?
7. What would you improve, automate, or monitor next time?

### Upload Rule

The final documented lab must be uploaded to the correct numbered topic folder.

Labs should not be uploaded to the repository root.
