# Reference Map

### Purpose

This file maps the LinuxLabs programme to the reference books and operational thinking sources.

### Core Thinking Sources

| Source | Used for |
| --- | --- |
| The Practice of System and Network Administration | Operational discipline, documentation, standardisation, service ownership, incident thinking |
| The Practice of Cloud System Administration | Reliability, automation, scalable operations, cloud-style administration, production readiness |

### Technical Reference Sources

| Source | Used for |
| --- | --- |
| The Linux Command Line | Shell, filesystem navigation, text processing, redirection, scripting basics |
| Learning Modern Linux | Modern Linux concepts, processes, filesystems, networking, observability, containers |
| Practical Linux System Administration | Installation, users, networking, package management, services, storage, administration tasks |
| Modern Operating Systems | Processes, memory, filesystems, I/O, concurrency, security, and OS theory behind Linux behaviour |
| Red Hat documentation | RHEL-specific implementation and administration guidance |
| Ubuntu documentation | Ubuntu-specific implementation and administration guidance |

### Distribution Rule

Labs should normally compare **RHEL** and **Ubuntu**.

If a lab cannot reasonably be completed on both distributions, the lab must explain why and record the limitation.

### Reference Usage Rule

Do not copy book material into the repository.

Use references to understand the task, then write original notes, evidence, and reflections.
